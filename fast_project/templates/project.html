Student Management System

For context, we're building a Student Management System for a university. This system handles various aspects of student data, including enrollment, courses, grades, and more. The system uses sql_Crab(ORM) to interact with the database, simplifying complex queries and operations without the need to write raw SQL.

Skip & View Implementation

Use Case: Managing Student Records

1. Student Enrollment

When a new student enrolls, their information is added to the system, including their name, date of birth, address, and contact details. The ORM maps this data to the Student table in the database, making it easy to add new students and retrieve their information later. 

            
            # Example
            class Student(Crabmodel):
                _column = {    
                    'first_name': DataTypes.varchar(max_length=20),
                    'last_name': DataTypes.varchar(max_length=20),
                    'address': DataTypes.varchar(max_length=30),
                    'email': DataTypes.emailfield(unique=True)
                }

2. Course Registration

Students can register for courses each semester. The ORM handles the relationships between students and the courses they are enrolled in, allowing easy retrieval of a student's course list or all students registered for a particular course. 

            
            # Example
            class Course(Crabmodel):
                _column = {
                    'course_name': DataTypes.varchar(max_length=100),
                    'course_code': DataTypes.varchar(max_length=10)
                }
